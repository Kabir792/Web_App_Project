# **Use Cases**

## **Use Case 1: User Login**

### **Actor**

Administrator, Teacher, Student

### **Description**

User enters login credentials to access the system.

### **Preconditions**

User account exists.

### **Main Flow**

1. User opens login page.  
2. User enters username and password.  
3. System validates credentials.  
4. User gains access to dashboard.

### **Postconditions**

User session is created.

---

## **Use Case 2: Add Student**

### **Actor**

Administrator

### **Description**

Administrator adds a new student record.

### **Main Flow**

1. Administrator selects Add Student.  
2. Administrator enters student information.  
3. System validates data.  
4. System saves student information.

### **Postconditions**

Student record is stored.

---

## **Use Case 3: Record Attendance**

### **Actor**

Teacher

### **Description**

Teacher records attendance for a class.

### **Main Flow**

1. Teacher selects course.  
2. Teacher marks attendance.  
3. System stores attendance records.

### **Postconditions**

Attendance data is updated.

---

## **Use Case 4: View Grades**

### **Actor**

Student

### **Description**

Student views academic grades.

### **Main Flow**

1. Student logs in.  
2. Student opens grades section.  
3. System displays grades.

### **Postconditions**

Student reviews academic performance.
