# **System Design**

## **Architecture**

The Student Management System follows a three-tier architecture.

### **Presentation Layer**

Responsible for:

* User Interface  
* Forms  
* Dashboards

Technologies:

* HTML  
* CSS  
* JavaScript

### **Application Layer**

Responsible for:

* Business Logic  
* Authentication  
* Validation

Technologies:

* PHP / Node.js / Java Spring Boot

### **Data Layer**

Responsible for:

* Data Storage  
* Data Retrieval

Technology:

* MySQL Database

---

## **System Flow**

1. User Login  
2. Authentication  
3. Dashboard Access  
4. Data Processing  
5. Database Operations  
6. Result Display
